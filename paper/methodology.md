# Methodology

## 1. Threat Model

**Adversary.** A passive, computationally bounded adversary who receives only the proxy text forwarded to the cloud LLM. The adversary cannot observe: (a) the local Gemma model weights or activations, (b) the entity_map substitution dictionary, (c) the DP noise seed, (d) the routing decision. The adversary may hold a large corpus of clinical documents from the same distribution to train inversion models.

**Attack surface.** Five attack classes are evaluated (§4). The primary threat model is the trained inversion attacker (Attack 3): given enough proxy–original pairs from prior interactions, can an adversary build a model that recovers sensitive token spans from proxy text alone?

**Out of scope.** Side-channel attacks against the local process, attacks exploiting the Anthropic API response, adaptive adversaries with knowledge of the NGSP internals, and real patient data are all explicitly out of scope.

## 2. System Architecture

The NGSP pipeline composes three defense layers:

### Layer 1 — Safe Harbor Stripper (`src/ngsp/safe_harbor.py`)

Deterministic regex + optional Gemma NER pass for the 18 HIPAA Safe Harbor identifiers (45 CFR §164.514(b)(2)). Returns a `StripResult` with:
- `stripped_text`: input with entities replaced by placeholders (`<PERSON_1>`, `<DATE_1>`, etc.)
- `entity_map`: dict mapping placeholder → original value (never leaves the local process)
- `spans`: list of `SensitiveSpan` objects with character offsets

### Layer 2 — Neural Router (`src/ngsp/router.py`)

Gemma 4 (`google/gemma-4-E2B-it`) classifies the stripped input into one of three paths:

| Path | Fraction (design target) | Handling |
|------|--------------------------|----------|
| `abstract_extractable` | ~70% | Query synthesizer generates a new self-contained question |
| `dp_tolerant` | ~20% | DP bottleneck with calibrated Gaussian noise |
| `local_only` | ~10% | Gemma answers locally; no cloud call made |

### Layer 3 — Path-Specific Privacy Mechanism

**Abstract-extractable path** (`src/ngsp/query_synthesizer.py`): Gemma generates a new question that captures task intent without referencing sensitive entities. The mapping is non-injective by construction: multiple inputs can produce the same synthesized query, making inversion statistically impossible even for an adversary with the proxy.

**DP-tolerant path** (`src/ngsp/dp_mechanism.py`): Hidden-state bottleneck at layer −1 of Gemma 4. Per-call mechanism:
1. Generate proxy text; pool hidden states → single vector `h ∈ ℝ^d`.
2. Clip to L2 norm C = 1.0: `h_clip = h · min(1, C/‖h‖₂)`.
3. Add Gaussian noise: `h_noisy = h_clip + N(0, σ²I)`.
4. Decode proxy via Gemma paraphrase conditioned on noisy hint.
5. Accumulate ε via Rényi DP accountant.

**DP calibration.** σ is set analytically:

```
σ = Δ · √(2 ln(1.25/δ)) / ε
```

where Δ = C = 1.0 (L2 sensitivity after clipping), δ = 1e-5, ε ∈ {0.5, 1.0, 2.0, 3.0, 5.0, 10.0}. Default target: ε = 3.0, giving σ ≈ 1.10.

**Rényi DP accounting.** Per-step cost R_α = α·Δ²/(2σ²). Conversion to (ε, δ)-DP: ε = R_α + log(1/δ)/(α−1). This follows the Rényi-DP accounting perspective introduced by Mironov (2017) and the practical subsampled accounting line developed by Wang et al. (2019). The `RDPAccountant` class tracks cumulative budget per session; calls are hard-refused once the cap is reached (`BudgetExhaustedError`).

### Answer Applier (`src/ngsp/answer_applier.py`)

Re-applies entity_map to the Anthropic response (longest-key-first to avoid partial-match collisions) before returning the final answer to the user.

## 3. Synthetic Corpus

All evaluation uses **synthetic data only**. The corpus is generated by `src/data/` with seed 42 (reproducible). Statistics:

| Document type | Generator | Target count | Sensitive categories |
|--------------|-----------|--------------|----------------------|
| SAE narratives | `synthetic_sae.py` | 500 | NAME, DATE, SITE, COMPOUND_CODE, AE_GRADE, EFFICACY_VALUE, … |
| Protocol excerpts | `synthetic_protocol.py` | 200 | DOSE, ELIGIBILITY_CRITERION, AMENDMENT_RATIONALE, … |
| Monitoring reports | `synthetic_monitoring.py` | 200 | SITE_ID, INVESTIGATOR_NAME, ENROLLMENT_STAT, … |
| CSR drafts | `synthetic_writing.py` | 300 | INTERIM_RESULT, REGULATORY_QUESTION, … |

Ground-truth sensitive spans are tracked at entity-slot insertion time; the `annotate(doc)` oracle in `src/data/annotator.py` returns the full `list[SensitiveSpan]` with precise character offsets.

Fake entity conventions:
- Names: `Dr. Testname Alpha`, `Patient Delta-7`, etc.
- Compound codes: `SYN-XXXX` format
- Site IDs: 9000-range integers
- Indications: blended synthetic vocabulary (e.g., "Synthofibrosis Type II")

## 4. Attack Suite

### Attack 1 — Verbatim (`src/attacks/verbatim.py`)

For each `(proxy_text, list[SensitiveSpan])` pair:
- **Literal check**: `span.value in proxy_text` (case-sensitive)
- **Fuzzy check**: `rapidfuzz.fuzz.partial_ratio(span.value, proxy_text) ≥ threshold` (default 85)

Metric: per-category leak rate and corpus-level `overall_literal_leak_rate`.

### Attack 2 — Cross-encoder Similarity (`src/attacks/similarity.py`)

Bi-encoder model: `all-MiniLM-L6-v2` (sentence-transformers). Cosine similarity between `encode(original)` and `encode(proxy)`. Metrics: `mean_sim`, `median_sim`, `p95_sim`, `fraction_above_threshold` (threshold = 0.85).

High similarity indicates the proxy preserves enough semantic content that a retrieval attack could link it to the original.

### Attack 3 — Trained Inversion (`src/attacks/inversion.py`)

Primary threat model. `DistilBertForTokenClassification` (distilbert-base-uncased) trained to predict per-token binary labels: 1 if the token overlaps any sensitive span in the original document, 0 otherwise.

- **Dataset split**: 80% train / 20% evaluation (seed-deterministic)
- **Training**: 3 epochs, AdamW lr = 2e-5, batch size 1, MPS/CPU device
- **Metric**: token-level F1 on held-out eval set
- **Random baseline**: predict positive with probability = corpus positive-token rate
- **Shuffle-pair control**: train on same proxies with labels from a different (shuffled) document; confirms learned signal is not spurious

Threshold for H₁: F1 ≤ 0.09.

### Attack 4 — Membership Inference (`src/attacks/membership.py`)

For each of the top-3 most-frequent compound codes in the corpus:
- **Positive set**: proxy embeddings from documents mentioning the entity
- **Negative set**: proxy embeddings from control documents
- **Classifier**: logistic regression on `all-MiniLM-L6-v2` embeddings (scikit-learn)
- **Metric**: AUC-ROC (expected ≈ 0.5 under strong privacy)

### Attack 5 — Utility Regression (`src/attacks/utility.py`)

For each document in a 50-doc evaluation sample:
1. Original path: send original text to Anthropic API with task prompt ("rewrite as a concise plain-language summary"); receive response.
2. Proxy path: run full NGSP pipeline; send proxy text to Anthropic API; receive response.
3. Judge both responses via Claude-as-judge with a scoring prompt (0.0 = useless, 1.0 = identical meaning).
4. `utility_ratio = mean(proxy_score) / mean(original_score)`.

All Anthropic API calls route through `RemoteClient` (canary detection + hash-only audit log enforced).

## 5. Library Versions

Verified via context7 MCP before implementation:

| Library | Version checked | Role |
|---------|----------------|------|
| `transformers` | 5.5.4 | Gemma 4 inference, DistilBERT fine-tune |
| `anthropic` | 0.96.0 | Anthropic API client |
| `huggingface_hub` | 1.11.0 | Model download (note: `resume_download` removed in v1.0) |
| `sentence-transformers` | current | Attacks 2 and 4 embeddings |
| `torch` | current stable | GPU/MPS tensor ops |
| `opacus` | current | Rényi DP reference (accountant inlined for session-scope control) |
| `rapidfuzz` | current | Attack 1 fuzzy matching |
| `scikit-learn` | current | Attack 4 logistic regression |

Gemma model: `google/gemma-4-E2B-it` (smallest released Gemma 4 instruct variant, ~2B parameters). Loaded via HF `transformers` (not Ollama/llama.cpp) because the DP bottleneck path requires `output_hidden_states=True` in `model.generate`; model selection follows the published Gemma 4 model card.

**MPS note**: Using `device_map="mps"` with `accelerate` causes a "Invalid buffer size: 9.51 GiB" pre-allocation failure on Apple Silicon. Workaround: load without `device_map`, then call `model.to("mps")` explicitly.

## 6. Reproducibility

- All corpus generators accept a `seed` parameter (default 42).
- All attack randomness is seeded: DistilBERT training (`torch.manual_seed`, `np.random.seed`, `random.seed`), membership inference train/test split, verbatim and similarity evaluations are deterministic.
- Model names and versions are recorded in every result JSON.
- Raw inputs and outputs are never logged; only SHA-256 hashes appear in `experiments/results/audit.jsonl`.
- Results JSON files are written to `experiments/results/`. The directory is excluded from `.gitignore` at the raw result level; summary files may be committed.
